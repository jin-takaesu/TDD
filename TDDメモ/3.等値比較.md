# 等値比較

前章副作用で `return` で Dollar オブジェクトを返すようにし、`times()` 関数は Dollar オブジェクトを受け取れるようになりました。
しかし現在その返ってきた値が正しいか確かめる方法がありません。
`EXPECT_EQ(10, ten.amount);` 変数 `amount` を直接参照しているので、将来 `amount` を非公開（private）にしたくなったときにテストが壊れてしまいます。

これを壊れないようにするにはそもそもオブジェクト同士で比較できれば内部構造を知らなくても確かめられるよね！ということです。

自分でも言っている意味が分からなくなってきたので簡単に言うと
「X=X」って正しいよね、だって「X=X」なんだもん。的な感じです。
X の中身なんか知らなくてもこれを実装してテストすれば等値か等値じゃないかわかるってことです。

---

## 現在の test.cpp

```cpp
TEST(DollarTest, Multiplication) {
    Dollar five(5);
    Dollar ten = five.times(2);

    EXPECT_EQ(10, ten.amount);  // ← こいつ

    Dollar fifteen = five.times(3);
    EXPECT_EQ(15, fifteen.amount);
}
```

こいつを

```cpp
EXPECT_EQ(10, ten.amount);
```

こう！

```cpp
EXPECT_TRUE(ten == Dollar(10));  // オブジェクト同士で比較する
```

もともとは「ten の amount は10ですか？」と聞いてるところを
「ten は Dollar(10) と同じですか？」と聞くことによって中身を知らなくても
Dollar と同じかどうかはわかるといった感じです。

ついでに `EXPECT_EQ(15, fifteen.amount);` も

```cpp
EXPECT_TRUE(fifteen == Dollar(15));
```

に書き換えてコンパイルすると、

```
error: no match for 'operator=='
```

といったエラーが。TDD なのでこれは正しい手順です。前進してます。Red を Green にする。これが TDD。

---

## Step 1: Red

`operator==` が未定義とブチギレられているので定義してあげます。

## Step 2: Green — Dollar.h に追記する

```cpp
class Dollar {
public:
    int amount;
    Dollar(int amount) : amount(amount) {}
    Dollar times(int multiplier) {
        return Dollar(amount * multiplier);
    }
    bool operator==(const Dollar& other) const {  // ← この辺
        return amount == other.amount;
    }
};
```

これで `operator==` を定義することでテストが通るようになります。

`bool` は真偽値の型で、引数 `(const Dollar& other)` は

- `const Dollar&` — 比較相手を参照で受け取る
- `other` — 比較相手のオブジェクト（右辺）
- 末尾の `const` — このメソッド自体が自分（this）の中身を変更しない定数

`return amount == other.amount;` で比較して `amount` 同士が等しければ `true` を返すという内容です。

これにより内部構造`amount`を直接参照せずにテストが書けるようになりました。
