# 三角測量

オブジェクト同士で比較することで中身の値を直接参照しなくても等値か不等値か確かめることができるようになりました。
しかし現在の状況じゃ信頼に値するようなテストはできていません。
なぜなら以下のテストしかしていないからです。

```cpp
EXPECT_TRUE(ten == Dollar(10));
EXPECT_TRUE(fifteen == Dollar(15));
```

こいつらは「等しい場合に true を返す」といった内容です。
これだと仮に以下のような常に true を返す実装でも通ってしまいます。

```cpp
bool operator==(const Dollar& other) const {
    return true;
}
```

三角測量の説明として、三角形の「1辺の長さ」と「両端の2つの角度」を計測し、そのデータから、離れた場所にある地点までの距離や位置（座標）を計算で求める測量技術です。主に地図作製、土地の測量、レーザーセンサーによる距離測定などに利用される、信頼性の高い手法です。

この算出方法をテストにも応用。
**「常に true を返す実装」だったとしたら、**
**二つ目の角度「必ず false になるテスト」を実装すればいいんです。**

```cpp
EXPECT_TRUE(ten == Dollar(10));
EXPECT_TRUE(fifteen == Dollar(15));
EXPECT_FALSE(Dollar(5) == Dollar(6));  // ← new
```

仮に上記の `return true;` みたいなどんな場合でも true を返すコードでも、新しく追加した `EXPECT_FALSE()` があれば true が返ってきてテストが通らなくなります。
つまり「その実装は本当に正しい実装なのか」みたいな疑いをかけ正当性を確認するテストコードというわけです。

この方法の考え方は

- 「等しいとき true」だけでは実装が正しいか確定できない
- 「等しくないとき false」も加えて2方向から確認して初めて確定できる

というものです。
なので自分が実装したものが甘ったれたこと書いてないかの証明として三角測量は非常に有効的な方法と言えます。歳をとっても自分のことを疑い続け時代に合った人間になりたいもんですね。

## プライバシー

前章のどこかで「将来 `amount` を非公開（private）にしたくなったときにテストが壊れてしまいます。」ということを言ったのですが、じゃあ具体的にどう実装するんだということをやっていきます。
そもそもなんで `private` にしたいんだって理由ですが意図しない書き換えが一番だと思っています。例えばデバッグの為に一時的に変更したのがそのまま残ったり、public だからクラスの外側で処理書いちゃっていっかなどの慢心だったりもあります。とにかく「うっかり」をちゃんとブロックしてもらうためにも `private` にする。といった感じです。

では、実装についてですが簡単です。現在 `amount` は `public` になっていますので `private` にするだけです。

**変更前の Dollar.h**

```cpp
class Dollar {
public:
    int amount;  // 誰でも読み書きできる
    ...
};
```

**変更後の Dollar.h**

```cpp
class Dollar {
private:
    int amount;  // Dollar クラスだけが触れる
public:
    ...
};
```

これでアクセスが制限でき、クラスがデータを守れるようになります。いわゆるカプセル化ですね。
前章で追加した `bool operator==(){}` もクラスのメンバ関数なのでエラーにならずに通ります。
